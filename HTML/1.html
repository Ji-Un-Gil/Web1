h1~6 : 제목을 의미하는 태그
p, br : 문장 바꿈을 의미하는 태그, p로 표현하는 것이 단락을 이해하는 데 도움이 되므로 가급적 p 사용 / br은 대신 줄바꿈을 계속 할 수 있는 장점
strong, u : 진하게, 언더바(밑줄)
img : 이미지 파일, 사이트주소를 이용 or 작업폴더에 파일을 직접 넣어 해당 파일을 바로 불러오는 방법 존재
li, ul, ol : li : 목차, 부모 자식관계에 있는 태그(예.li 태그를 안 쓰면 ul은 쓸 필요없는 태그가 됨), ul : 목록 구분, ol : 자동 번호 매김
title : 사이트 제목을 의미하는 태그, 검색엔진이 중요하게 생각함, 글자깨짐 현상 > 문자 표현 방식이 웹페이지와 코딩페이지가 달라 충돌하는 현상
meta charset="utf-8" : 이렇게 웹사이트가 우리가 쓴 문자표현 방식을 이해하도록 문장 추가가 필요!!
head, body, html : head : 본문을 설명하는 부분(제목부분), body : 본문부분, html : head와 body를 감싸는.. 이렇게 약속한 것
!DOCTYPE html : 이 코딩이 html로써 만들어졌음을 의미
a href="사이트주소" target="_blank" title="blabla" : a href : 링크를 의미, target="_blank" : 새창에서 링크가 열리게함, title="" : 링크가 담는 내용을 툴팁(링크에 갖다댔을 때 뜨는 팝업)으로 보여줌
